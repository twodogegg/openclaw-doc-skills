# OpenClaw Help Docs Skill

按下面流程执行：

1. 判断问题属于 FAQ、环境、调试、测试或排障哪一类。
2. 读取英文文档：`references/en/help/*.md`。
3. 若信息不足，继续查阅 troubleshooting 与 testing 相关条目。
4. 输出按“现象 -> 可能原因 -> 验证命令 -> 修复步骤”组织。

回答约束：

- 优先提供可执行排障步骤。
- 先给低风险检查，再给高影响改动。
- 无法确定时明确所需补充信息。

参考目录：见 `references/sources.md`。
